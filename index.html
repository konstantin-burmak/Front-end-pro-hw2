ДЗ 2. Операторы и типы данных

1. false && true || true; // операторы И, ИЛИ вычисляют операнды слева на право. В таком случае первое сравнение && вернет значение false, так как И возрвращает true, если оба аргумента истинны. Второе сравнение || вернёт значение true, так как ИЛИ возрвращает true, если какой-либо из аргументов true. Ответ - true.

2. false && (true || true); // По приоритету сначала вычисляются операции в скобках, вернёт значение true. после сравнение && вернёт значение false. Ответ - false.

3. +'40' + +'2'; //Так как унарный плюс преобразовывает строки в числа и бинарный плюс складывает эти два числа, то Ответ - 42.

4. '2' + 3 ** 2; // приортет оператора возведения в степень выше чем сложения, поэтому сначала возводим в степень и получаем число 9, а после складываем, и так как первый операнд является строкой, то второй также будет преобразован в строку. Ответ - '29'

5. 3 ** 2 / 3; // Возведение в степень имеет приоритет выше, поэтому сначала возводим в степень, получаем число 9, после этого выполняем деление. Ответ - 3

6. 3 ** (9 / 3); // По приоритету сначала выполняется деление в скобках, после возводим в степень. Ответ - 27

7. '10' + (5 === 6); // Сначала в скобках выполняем действия строгого равенства, получаем значение false, так как они не равны. После этого складываем, и так как первый операнд является строкой, то остальные также преобразуются в строку. Ответ - '10false'

8. '10' - 5 === 6; // Сначала производим проверку строгого равенства, получаем false. После этого производим вычитаение, а так как пробразование строк - это особенность бинарного плюса, то вычитание преобразовывают операнды в числа. Но так как оператор строгого равенства проверяет без преобразования типов, то проверка вернёт false. Так как это разные типы. Ответ - false

9. undefined + 1; // Так как при использовании математических операторов undefined преобразовывается в NaN, то Ответ - NaN

10. (+null == false) < 1; // Так как при использовании математических операторов null становится 0, то сравнение в скобках вернёт true, так как 0 и false - одно и то же. А так как true и 1 также одно и то же, 1 не может быть больше 1. Поэтому при проверке получим Ответ - false

11. (+null == false) < 1 ** 5; // Как и в предыдущем примере в скобках получаем значение true. После возводим в степень, получаем число 1. В итоге true преобразовывается в число 1 и получаем Ответ - false

var number = 5; // Обьявляем переменную.

2 * ++number; // Префиксная форма инкримента вернёт новое значение, то есть 6. При умножении получим число 12.

2 * number--; // Постфиксная форма дикримента вернёт старое значение, то есть 5. При умножении получим число 10.

console.log(number); // Так как переменная number не переопределена, выводится изначальное её значение.
